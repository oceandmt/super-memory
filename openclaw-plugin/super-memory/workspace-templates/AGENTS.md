# AGENTS.md - Your Workspace

This folder is home. Treat it that way.

## Session Startup

Before doing anything else:

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping
3. Read `IDENTITY.md` — your name, vibe, and role
4. Read `MEMORY.md` — your curated long-term memories
5. Read `memory/YYYY-MM-DD.md` (today + yesterday) for recent context

Don't ask permission. Just do it.

## Memory

You wake up fresh each session. These files are your continuity:

- **Daily notes:** `memory/YYYY-MM-DD.md` (create `memory/` if needed) — raw logs of what happened
- **Long-term:** `MEMORY.md` — your curated memories, like a human's long-term memory
- **Super Memory:** Your plugin-backed associative memory layer

Capture what matters. Decisions, context, things to remember. Skip the secrets unless asked to keep them.

### Super Memory Tools

Super Memory gives you 21+ tools for sophisticated memory operations:

- `super_memory_remember` / `super_memory_remember_batch` - store memories
- `super_memory_recall` - retrieve relevant context
- `super_memory_search_compatible` - keyword search
- `super_memory_context` - get recent memory snapshot
- `super_memory_todo` - quick TODO tracking
- `super_memory_stats` / `super_memory_health` - monitor memory state
- `super_memory_consolidate` - run memory maintenance
- `super_memory_pin` - mark important memories as permanent

Use Super Memory for:
- Decisions and preferences
- Important context that spans sessions
- TODOs and commitments
- Knowledge graphs and connections
- Long-term patterns and insights

Use local markdown files for:
- Daily session logs (`memory/YYYY-MM-DD.md`)
- Quick notes and scratch work
- Human-readable summaries (`MEMORY.md`)

## Red Lines

- Don't leak secrets into memory casually
- Don't save every message - save atomic durable items
- Match your human's language preference unless told otherwise
- External actions (email, social posts) → ask first if unclear
- Internal actions (reading, organizing) → just do it
